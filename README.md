# 高等数学知识图谱 · 可视化系统 — 使用说明

> [!IMPORTANT]
> **声明：** 本项目仅作为一个演示 Demo（演示原型），用于展示图谱交互及数据管理的基本流程。

## 一、项目简介

本项目是一个基于 **Next.js** 的交互式知识图谱可视化应用，以高等数学核心知识为示例数据，使用 **D3.js** 力导向图（Force-Directed Graph）呈现节点与关系。

默认包含 20 个知识节点和 38 条关系，涵盖函数、极限、导数、积分、级数等高等数学核心概念及其关联。

---

## 二、快速启动（从零开始，手把手教你跑起来）

> [!NOTE]
> 前提：你的电脑需要安装 **Node.js**（版本 18 以上）。
> 如果没有，去 [nodejs.org](https://nodejs.org) 下载安装，安装完成后，打开终端输入 `node -v`，若能看到版本号则说明安装成功。

### 第 1 步：进入项目文件夹
打开终端（Windows 的 PowerShell 或 CMD，macOS 的 Terminal），进入项目根目录：
```bash
cd D:\Downloads\knowledge-graph-source
```

### 第 2 步：安装项目依赖（仅首次运行需要）
```bash
bun install
```
> [!TIP]
> 这一步会自动下载项目需要的所有代码库，可能需要几十秒。
> 如果提示 `"bun 不是命令"`，请先安装 Bun：
> - **Windows** (在 PowerShell 中运行)：
>   ```powershell
>   powershell -c "irm bun.sh/install.ps1 | iex"
>   ```
> - **macOS/Linux**：
>   ```bash
>   curl -fsSL https://bun.sh/install | bash
>   ```
> 安装完成后关闭终端重新打开，再执行 `bun install`。

### 第 3 步：初始化数据库（仅首次运行需要）
```bash
bun run db:push
bun run db:seed
```
- 第一条命令创建数据库和表结构。
- 第二条命令写入默认的 20 个知识节点和 38 条关系。

### 第 4 步：启动项目
```bash
bun run dev
```
- 启动后看到 `"Ready in XXXms"` 的提示，说明服务已成功启动。
- 运行期间请勿关闭终端窗口。如需停止服务，请在终端按 `Ctrl + C`。

### 第 5 步：打开浏览器查看效果
在浏览器地址栏输入: **[http://localhost:3000](http://localhost:3000)**。
推荐使用 Chrome / Edge / Firefox 浏览器。

---

### 常见问题

- **页面打开后是空白？**
  数据加载需要时间，请稍等几秒。如果一直空白，请按 `F12` 打开浏览器控制台查看报错信息。
- **提示“端口 3000 被占用”？**
  说明已有其他程序占用该端口。请关闭占用 3000 端口的其他程序或服务后重试。
- **`bun install` 报错？**
  请检查网络连接是否正常。如果网络不好，可以尝试使用 `npm install`。
- **我想清空所有数据重新开始？**
  删除项目根目录下 `prisma` 文件夹里的 `dev.db` 文件，然后重新执行第 3 步和第 4 步。

## 三、界面功能说明

### 【顶部工具栏】
- **搜索框**：输入关键词，实时高亮匹配的节点。
- **添加节点**：手动添加单个节点（只需填写名称，ID 自动生成）。
- **添加关系**：在已有节点间添加关系。
- **批量导入**：通过 Markdown 格式批量导入节点和关系（详见第四节）。
- **导出HTML**：将当前图谱导出为离线单文件 HTML（内嵌 D3.js，可完全离线浏览）。
- **重置**：将数据恢复为默认的 20 节点 38 关系（需要弹窗确认）。
- **节点/关系计数**：实时显示当前图谱中的节点和关系总数。

### 【图谱交互】
- **拖拽节点**：鼠标按住节点拖动可手动调整位置。
- **滚轮缩放**：通过鼠标滚轮控制图谱放大或缩小。
- **悬停高亮**：鼠标悬停于某一节点时，高亮该节点及其直接关联的节点和连线，其余部分变暗。
- **点击详情**：点击节点后，右侧面板会展示该节点的详细属性、关联节点及关系。
- **取消选中**：点击图谱空白区域即可取消选中。

### 【左侧边栏】
- **实体类型筛选**：点击可切换显示或隐藏“概念/定理/方法/应用”四种类型节点。
- **关系类型列表**：显示当前图谱中存在的所有关系类型及各自数量。
- **统计信息**：显示当前节点总数和关系总数。

### 【右侧详情面板】
- **节点信息**：名称、类型、描述。
- **编辑节点**：点击铅笔图标编辑节点名称、类型和描述。
- **关联节点**：展示所有相连节点，支持点击跳转。
- **关联关系**：展示所有相关的连接关系，可单独执行删除。
- **删除节点**：点击红色删除按钮，删除该节点时会级联删除其所有关联关系（需确认）。

## 四、批量导入（Markdown 格式）

点击顶部工具栏的 **「批量导入」** 按钮，在文本框中粘贴符合规范的 Markdown 格式数据即可。

### 【格式规范】

1. **用标题标记节点类型（中英文均支持）：**
   - `# 概念` 或 `# concept`
   - `# 定理` 或 `# theorem`
   - `# 方法` 或 `# method`
   - `# 应用` 或 `# application`

2. **节点定义（列表格式，冒号支持中英文冒号）：**
   - `- 节点名称: 节点描述` （描述可省略）
   - `- 节点名称：节点描述`
   - `- 节点名称` （省略描述的写法）

3. **关系定义（在 `# 关系` 或 `# relationships` 标题下）：**
   - `- 源节点 --关系名--> 目标节点`

### 【完整示例】
```markdown
# 概念
- 函数: 描述两个变量之间依赖关系的基本概念
- 极限: 描述函数在无限接近过程中变化趋势的重要概念
- 导数: 描述函数变化率的工具

# 定理
- 罗尔定理: 闭区间上连续开区间内可导且端点值相等的函数，必存在导数为零的点
- 拉格朗日中值定理: 闭区间上连续开区间内可导的函数，必存在切线平行于割线的点

# 方法
- 求极限方法: 包括洛必达法则、等价无穷小替换等多种求极限的技巧

# 应用
- 物理应用: 微积分在物理学中的应用
- 几何应用: 微积分在几何学中的应用

# 关系
- 导数 --基于--> 极限
- 微分 --基于--> 导数
- 积分 --互逆--> 导数
- 级数 --基于--> 极限
```

### 【注意事项】
- 节点 ID 等于节点名称（冒号前的部分），在同一批次中不能重复定义。
- 关系中的源节点和目标节点必须在节点定义中出现过，或者已在数据库中存在。
- 若关系中引用了但未在标题下定义的节点，系统会自动创建为「概念」类型。
- 重复导入已存在的节点和关系会被系统自动跳过。

| :--- | :---: | :---: | :--- |

## 六、八种关系类型说明

## 七、导出离线 HTML

点击 **「导出HTML」** 按钮，会下载一个 `knowledge-graph.html` 文件。

**特点：**
- 单文件，无需网络即可直接双击运行（D3.js 库已内嵌）。
- 包含当前数据库中的所有节点和关系快照数据。
- 支持完整的交互功能（拖拽、缩放、悬停高亮等）。

## 八、Electron 桌面应用打包

本项目支持通过 Electron 打包为独立的桌面应用程序（`.exe` / `.dmg` / `.AppImage`）。

### 1. 开发模式（Electron 容器包裹）
```bash
# 启动本地服务
bun run dev

# 启动 Electron 开发窗口（在另一个终端执行）
bun run electron:dev
```

### 2. 打包为桌面应用
根据平台运行对应的打包脚本：
```bash
# 打包当前平台应用
bun run electron:build

# 打包 Windows .exe
bun run electron:build:win

# 打包 macOS .dmg
bun run electron:build:mac

# 打包 Linux .AppImage
bun run electron:build:linux
```
打包输出路径为：`dist-electron/`。

### 3. 特性与注意事项
- 数据库位于用户数据目录，软件更新或重装不会丢失：
  - Windows: `%APPDATA%/knowledge-graph-math/db/`
  - macOS: `~/Library/Application Support/knowledge-graph-math/db/`
- 首次打包会下载 Electron 依赖文件，时间可能较长。
- 跨平台打包需要在对应的操作系统平台下进行。

## 九、API 接口说明

| 请求方式 | 接口地址 | 说明 |
| :--- | :--- | :--- |
| **GET** | `/api/graph` | 获取所有节点和关系数据 |
| **POST** | `/api/graph/nodes` | 添加单个节点 |
| **PUT** | `/api/graph/nodes` | 编辑节点 |
| **DELETE**| `/api/graph/nodes?id=xxx` | 删除节点（级联删除其所有关联关系） |
| **POST** | `/api/graph/edges` | 添加单条关系 |
| **DELETE**| `/api/graph/edges?sourceId=x&targetId=x&relation=x` | 删除单条关系 |
| **POST** | `/api/graph/import` | 批量导入（Markdown 解析） |
| **POST** | `/api/graph/reset` | 重置为默认数据 |
| **GET** | `/api/graph/export` | 导出离线 HTML 文件 |

## 十、技术栈

- **框架**：Next.js 16 (App Router)
- **语言**：TypeScript 5
- **样式**：Tailwind CSS 4 + shadcn/ui
- **图谱渲染**：D3.js v7 力导向图
- **数据库**：Prisma ORM + SQLite
- **桌面打包**：Electron + electron-builder
