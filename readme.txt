          高等数学知识图谱 · 可视化系统 — 使用说明

一、项目简介
────────────────────────────────────────────────────────────────────────────────

  本项目是一个基于 Next.js 的交互式知识图谱可视化应用，以高等数学核心知识为
  示例数据，使用 D3.js 力导向图（Force-Directed Graph）呈现节点与关系。

  默认包含 20 个知识节点和 38 条关系，涵盖函数、极限、导数、积分、级数等
  高等数学核心概念及其关联。


二、快速启动（从零开始，手把手教你跑起来）
────────────────────────────────────────────────────────────────────────────────

  ▶ 前提：你的电脑需要安装 Node.js（版本 18 以上）
         如果没有，去 https://nodejs.org 下载安装，一路点"下一步"即可
         安装完成后，打开终端（Windows 按 Win+R 输入 cmd，Mac 按 Cmd+空格搜 终端）
         输入 node -v 回车，能看到版本号就说明安装成功

  ────────────────────────────────────────
  第 1 步：打开终端，进入项目文件夹
  ────────────────────────────────────────

     cd /你下载本项目的路径/my-project

     提示：把上面的路径换成你电脑上项目文件夹的实际位置
           例如 Windows: cd C:\Users\张三\Downloads\my-project
           例如 Mac:    cd ~/Downloads/my-project

  ────────────────────────────────────────
  第 2 步：安装项目依赖（首次运行必须执行，只需执行一次）
  ────────────────────────────────────────

     bun install

     💡 这一步会自动下载项目需要的所有代码库，可能需要几十秒
     💡 如果提示"bun 不是命令"，请先安装 Bun：
        Windows: 在 PowerShell 中运行
          powershell -c "irm bun.sh/install.ps1 | iex"
        Mac/Linux: 在终端中运行
          curl -fsSL https://bun.sh/install | bash
        安装完成后关闭终端重新打开，再执行 bun install

  ────────────────────────────────────────
  第 3 步：初始化数据库（首次运行必须执行，只需执行一次）
  ────────────────────────────────────────

     bun run db:push
     bun run db:seed

     💡 第 1 条命令创建数据库和表结构
     💡 第 2 条命令写入默认的 20 个知识节点和 38 条关系
     💡 如果看到"Done"或"Seed complete"提示，说明成功了
     💡 注意：即使忘记执行 db:seed，首次打开浏览器访问时也会自动写入默认数据

  ────────────────────────────────────────
  第 4 步：启动项目
  ────────────────────────────────────────

     bun run dev

     💡 启动后会看到类似 "Ready in XXXms" 的提示，说明服务已启动
     💡 运行期间不要关闭这个终端窗口，关闭 = 停止服务
     💡 如果要停止服务，按 Ctrl + C

  ────────────────────────────────────────
  第 5 步：打开浏览器查看效果
  ────────────────────────────────────────

     在浏览器地址栏输入: http://localhost:3000

     🎉 你应该能看到一个高等数学知识图谱，里面有彩色节点和连线！
     💡 推荐使用 Chrome / Edge / Firefox 浏览器

  ────────────────────────────────────────
  常见问题
  ────────────────────────────────────────

     问：页面打开后是空白？
     答：等几秒钟，数据加载需要时间。如果一直空白，按 F12 打开浏览器控制台
         看有没有红色报错。

     问：提示"端口 3000 被占用"？
     答：说明已经有程序在使用 3000 端口。关闭其他占用该端口的程序后重试。

     问：bun install 报错？
     答：检查网络连接是否正常。如果网络不好，可以尝试换成 npm install。

     问：我想重新开始，清空所有数据？
     答：删除项目根目录下 prisma 文件夹里的 dev.db 文件，
         然后重新执行第 3 步和第 4 步。


三、界面功能说明
────────────────────────────────────────────────────────────────────────────────

  【顶部工具栏】

    · 搜索框 — 输入关键词，实时高亮匹配的节点
    · 添加节点 — 弹出对话框，手动添加单个节点（只需填写名称，ID 自动生成）
    · 添加关系 — 弹出对话框，在已有节点间添加关系
    · 批量导入 — 通过 Markdown 格式批量导入节点和关系（详见第四节）
    · 导出HTML — 将当前图谱导出为离线单文件 HTML（内联 D3.js，可离线浏览）
    · 重置 — 将数据恢复为默认的 20 节点 38 关系（需确认）
    · 节点/关系计数 — 显示当前图谱中的节点和关系总数

  【图谱交互】

    · 拖拽节点 — 鼠标按住节点拖动可调整位置
    · 滚轮缩放 — 鼠标滚轮控制图谱缩放
    · 悬停高亮 — 鼠标悬停节点时，高亮该节点及其关联节点和边，其余变暗
    · 点击详情 — 点击节点后，右侧面板显示节点详情、关联节点和关联关系
    · 取消选中 — 点击图谱空白区域可取消选中

  【左侧边栏】

    · 实体类型筛选 — 点击切换显示/隐藏 概念/定理/方法/应用 四种类型
    · 关系类型列表 — 显示当前图谱中存在的所有关系类型及数量
    · 统计信息 — 显示节点总数和关系总数

  【右侧详情面板】

    · 节点信息 — 名称、类型、描述
    · 编辑节点 — 点击铅笔图标可编辑节点名称、类型和描述
    · 关联节点 — 可点击跳转到关联节点
    · 关联关系 — 显示与该节点相关的所有关系，可单独删除某条关系
    · 删除节点 — 红色删除按钮，删除前会弹出确认，删除节点会级联删除其关联关系


四、批量导入（Markdown 格式）
────────────────────────────────────────────────────────────────────────────────

  点击顶部工具栏的「批量导入」按钮，在文本框中粘贴 Markdown 格式数据即可。

  【格式规范】

    1. 用标题标记节点类型（中文和英文均支持）：

       # 概念 / # concept   → 概念类型节点
       # 定理 / # theorem   → 定理类型节点
       # 方法 / # method    → 方法类型节点
       # 应用 / # application → 应用类型节点

    2. 节点定义（列表格式，冒号支持中文「：」和英文「:」）：

       - 节点名称: 节点描述          （描述可省略）
       - 节点名称：节点描述          （中文冒号也可以）
       - 节点名称                     （省略描述的写法）

    3. 关系定义（在「# 关系」或「# relationships」标题下）：

       - 源节点 --关系名--> 目标节点

  【完整示例】

    # 概念
    - 函数: 描述两个变量之间依赖关系的基本概念
    - 极限: 描述函数在无限接近过程中变化趋势的重要概念
    - 导数: 描述函数变化率的工具

    # 定理
    - 罗尔定理: 闭区间上连续开区间内可导且端点值相等的函数，必存在导数为零的点
    - 拉格朗日中值定理: 闭区间上连续开区间内可导的函数，必存在切线平行于割线的点

    # 方法
    - 求极限方法: 包括洛必达法则、等价无穷小替换等多种求极限的技巧

    # 应用
    - 物理应用: 微积分在物理学中的应用
    - 几何应用: 微积分在几何学中的应用

    # 关系
    - 导数 --基于--> 极限
    - 微分 --基于--> 导数
    - 积分 --互逆--> 导数
    - 级数 --基于--> 极限

  【注意事项】

    · 节点 ID 等于节点名称（冒号前的部分），同一批次中不能重复定义
    · 冒号支持半角「:」和全角「：」两种写法
    · 关系中的源节点和目标节点必须在节点定义中出现过，或在数据库中已存在
    · 关系中引用了但未在标题下定义的节点，会自动创建为「概念」类型
    · 已存在的节点和关系会被自动跳过，不会报错


五、四种实体类型说明
────────────────────────────────────────────────────────────────────────────────

    类型      图标    颜色       说明
    ─────────────────────────────────────────────────
    concept   💡     蓝色       概念：学科基本概念和核心术语
    theorem   📜     绿色       定理：经过证明的数学命题
    method    🔧     黄色       方法：解决问题的具体技巧和步骤
    application 🎯   红色       应用：在实际场景中的运用


六、八种关系类型说明
────────────────────────────────────────────────────────────────────────────────

    关系名    线条样式          含义
    ─────────────────────────────────────────────────
    基于      实线（靛蓝色）    A 的理论基础是 B
    互逆      虚线（紫色）      A 与 B 互为逆运算
    研究      实线（蓝色）      A 研究的对象是 B
    定理      实线（绿色）      A 概念对应的定理是 B
    方法      实线（黄色）      A 概念对应的方法是 B
    应用      实线（红色）      A 概念的应用领域是 B
    支撑      短虚线（绿色）    A 定理支撑的方法是 B
    实现      短虚线（橙色）    A 方法实现的应用是 B


七、导出离线 HTML
────────────────────────────────────────────────────────────────────────────────

  点击「导出HTML」按钮，会下载一个 knowledge-graph.html 文件。

  特点：
    · 单文件，无需网络即可运行（D3.js 已内联）
    · 包含当前数据库中的所有节点和关系数据
    · 支持拖拽、缩放、悬停高亮等交互功能
    · 可直接发送给他人，用浏览器打开即可查看


八、Electron 桌面应用打包
────────────────────────────────────────────────────────────────────────────────

  本项目支持通过 Electron 打包为独立的桌面应用程序（.exe / .dmg / .AppImage），
  实现双击即用，无需安装 Node.js 或浏览器。

  【打包前提】

    1. 确保项目依赖已安装：bun install
    2. 确保数据库已初始化：bun run db:push && bun run db:seed
    3. 确保 Electron 已安装（已包含在 devDependencies 中）

  【开发模式（Electron 包裹开发服务器）】

     # 先启动 Next.js 开发服务器
     bun run dev

     # 在另一个终端启动 Electron
     bun run electron:dev

     💡 开发模式下，Electron 会连接到 localhost:3000 的开发服务器
     💡 支持热更新，修改代码后自动刷新

  【打包为桌面应用】

     # 打包当前平台的桌面应用
     bun run electron:build

     # 打包 Windows .exe
     bun run electron:build:win

     # 打包 macOS .dmg
     bun run electron:build:mac

     # 打包 Linux .AppImage
     bun run electron:build:linux

  【打包产物】

    打包完成后，安装包位于 dist-electron/ 目录：

    · Windows: dist-electron/高等数学知识图谱 Setup 1.0.0.exe
    · macOS:   dist-electron/高等数学知识图谱-1.0.0.dmg
    · Linux:   dist-electron/高等数学知识图谱-1.0.0.AppImage

  【桌面应用特性】

    · 双击运行，无需安装 Node.js 或浏览器
    · 数据库存储在用户数据目录（不会丢失）：
      - Windows: %APPDATA%/knowledge-graph-math/db/
      - macOS:   ~/Library/Application Support/knowledge-graph-math/db/
      - Linux:   ~/.config/knowledge-graph-math/db/
    · 首次运行自动从应用包复制初始数据库
    · 应用菜单提供「文件 > 重置数据」功能
    · 支持键盘快捷键（Ctrl+Shift+R 重置数据）

  【注意事项】

    · 打包体积约 150-300MB（内含 Chromium + Node.js 运行时）
    · 跨平台打包需要在对应平台上进行（Windows 打包在 Windows 上，以此类推）
    · 打包前会自动执行 Next.js 构建，无需手动 build
    · 首次运行打包命令可能需要下载 Electron 二进制文件，请耐心等待
    · 打包脚本会自动执行 db:seed，确保打包的数据库包含默认数据

  【Windows 常见问题】

    · 如果 PowerShell 报错"无法加载 npx.ps1，禁止运行脚本"：
      方法一：使用 CMD（命令提示符）代替 PowerShell 运行命令
      方法二：在 PowerShell 中执行以下命令允许脚本运行：
        Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser


九、API 接口说明
────────────────────────────────────────────────────────────────────────────────

  GET    /api/graph                                  获取所有节点和关系
  POST   /api/graph/nodes                            添加单个节点
  PUT    /api/graph/nodes                            编辑节点
  DELETE /api/graph/nodes?id=xxx                     删除节点（级联删除关联关系）
  POST   /api/graph/edges                            添加单条关系
  DELETE /api/graph/edges?sourceId=x&targetId=x&relation=x  删除关系
  POST   /api/graph/import                           批量导入（Markdown 格式）
  POST   /api/graph/reset                            重置为默认数据
  GET    /api/graph/export                           导出离线 HTML 文件


十、技术栈
────────────────────────────────────────────────────────────────────────────────

    框架       Next.js 16 (App Router)
    语言       TypeScript 5
    样式       Tailwind CSS 4 + shadcn/ui
    图谱渲染   D3.js v7 力导向图
    数据库     Prisma ORM + SQLite
    动画       Framer Motion
    桌面打包   Electron + electron-builder


十一、项目结构
────────────────────────────────────────────────────────────────────────────────

    prisma/
      schema.prisma         数据库模型定义（KGNode, KGEdge）
      seed.ts               初始数据种子脚本
    src/
      app/
        page.tsx            主页面（图谱可视化 + 交互界面）
        api/graph/
          route.ts          GET 获取图谱数据
          nodes/route.ts    POST/PUT/DELETE 节点操作
          edges/route.ts    POST/DELETE 关系操作
          import/route.ts   POST 批量导入（Markdown 解析）
          export/route.ts   GET 导出离线 HTML
          reset/route.ts    POST 重置为默认数据
      components/
        force-graph.tsx     D3.js 力导向图组件
        ui/                 shadcn/ui 组件库
      lib/
        db.ts               Prisma 数据库客户端
        knowledge-graph-data.ts  类型定义和配置数据
    electron/
      main.js               Electron 主进程（窗口管理、服务器启动）
      preload.js            Electron 预加载脚本（安全桥接）
      icon.png              应用图标
    public/
      knowledge-graph.html  导出的离线 HTML 模板
